# awesome 宣言

如果你希望你的列表被收录在 `awesome` 中，请尽量只包含真正 awesome 的内容。毕竟这是精选，而不是收藏。

但 **什么才是 awesome？**

## 只有 awesome 才是 awesome

研究你要包含的内容是否真正 awesome。只将你或其他贡献者可以个人推荐的内容列入列表。宁可遗漏也不要包含过多。

## Awesome 徽章

以下是 Awesome 列表使用的徽章。

[![Awesome](https://awesome.re/badge.svg)](https://awesome.re)
[![Awesome](https://awesome.re/badge-flat.svg)](https://awesome.re)
[![Awesome](https://awesome.re/badge-flat2.svg)](https://awesome.re)

在列表顶部标题旁添加一个 awesome 徽章。[示例](https://github.com/sindresorhus/awesome-nodejs)。你可以选择常规徽章或扁平徽章。

```md
[![Awesome](https://awesome.re/badge.svg)](https://awesome.re)
[![Awesome](https://awesome.re/badge-flat.svg)](https://awesome.re)
[![Awesome](https://awesome.re/badge-flat2.svg)](https://awesome.re)
```

你可以在未收录在此处以及私有列表上使用该徽章。

*徽章不得以任何方式修改。*

## Awesome 提及徽章

此徽章适用于被收录在 Awesome 列表中的项目（**不是 Awesome 列表本身**）。例如，[Chalk](https://github.com/chalk/chalk) 项目可以使用此徽章，因为它被列在 [Awesome Node.js](https://github.com/sindresorhus/awesome-nodejs) 中。项目使用此徽章完全是可选的，但这是展示项目被收录在 Awesome 列表中的好方式。你可以选择常规徽章或扁平徽章。

[![Mentioned in Awesome](https://awesome.re/mentioned-badge.svg)](https://awesome.re)
[![Mentioned in Awesome](https://awesome.re/mentioned-badge-flat.svg)](https://awesome.re)

**填写占位符（名称和 URL）：**

```md
[![Mentioned in Awesome <列表名称>](https://awesome.re/mentioned-badge.svg)](https://github.com/<列表 URL>)
[![Mentioned in Awesome <列表名称>](https://awesome.re/mentioned-badge-flat.svg)](https://github.com/<列表 URL>)
```

示例：

```md
[![Mentioned in Awesome Node.js](https://awesome.re/mentioned-badge.svg)](https://github.com/sindresorhus/awesome-nodejs)
```

如果你是 Awesome 列表的维护者，可以鼓励列表中的项目添加此徽章。

*徽章不得以任何方式修改。*

## 说明为什么某样东西很 awesome

除了推荐列表中的特定项目，你还应该告诉读者它*为什么*在列表中，以及他们将如何从中受益。

## 明确列表的内容

在 readme 顶部有一个简洁的描述。确保你的列表涵盖特定范围，不涉及无关内容。如果认为其他 awesome 列表已经充分涵盖了某个主题，请链接到它们。

## 注意语法

确保你的列表语法正确、没有错别字，并且没有 Markdown 格式错误。这同样适用于 pull request。

## 选择合适的许可

请记住，如果你[没有选择许可](https://choosealicense.com/no-license/)，基本上意味着人们*不允许*复制、分发或创建衍生作品。

[Creative Commons 许可](https://creativecommons.org/)非常适合此目的。**我们建议使用 [`CC0`](https://creativecommons.org/publicdomain/zero/1.0/)。** MIT、BSD、GPL 等代码许可不予推荐。

## 包含贡献指南

贡献你列表的人应该清楚了解应该如何贡献。

如果你不想从头编写，可以随意使用我们的 [contributing.md](contributing_zh.md) 并根据自己的需求修改。

## 适当地设计你的列表

创建[目录](https://github.com/sindresorhus/stuff/blob/main/toc-generators.md)，将内容组织到不同类别，并在适当时使用图片。确保所有条目一致。

## 接受他人的意见

如果你是列表的拥有者，尊重他人的意见。如果有许多用户不同意你的决定，请三思。
