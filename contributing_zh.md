# 贡献指南

请注意，本项目发布了[贡献者行为准则](code-of-conduct_zh.md)。参与本项目即表示你同意遵守其条款。

## 添加一个 awesome 列表

请确保你的 pull request 遵循[指南列表](pull_request_template_zh.md)。

## 创建你自己的 awesome 列表

要创建你自己的列表，请查看[说明](create-list_zh.md)。

## 向 awesome 列表添加内容

如果你有 awesome 的内容要贡献给 awesome 列表，请按以下步骤操作。

你需要一个 [GitHub 账号](https://github.com/join)！如果你不熟悉 GitHub，请[从这里开始](https://github.com/firstcontributions/first-contributions)。

1. 访问 awesome 列表的 GitHub 页面。例如：https://github.com/sindresorhus/awesome
2. 点击 `readme.md` 文件：![步骤 2 点击 Readme.md](https://cloud.githubusercontent.com/assets/170270/9402920/53a7e3ea-480c-11e5-9d81-aecf64be55eb.png)
3. 现在点击编辑图标。![步骤 3 - 点击编辑](https://cloud.githubusercontent.com/assets/170270/9402927/6506af22-480c-11e5-8c18-7ea823530099.png)
4. 你可以在浏览器内的编辑器中开始编辑文件。确保遵循上述指南。你可以使用 [GitHub 风格的 Markdown](https://help.github.com/articles/github-flavored-markdown/)。![步骤 4 - 编辑文件](https://cloud.githubusercontent.com/assets/170270/9402932/7301c3a0-480c-11e5-81f5-7e343b71674f.png)
5. 说明你为什么提议更改，然后点击 "Propose file change"。![步骤 5 - 提议更改](https://cloud.githubusercontent.com/assets/170270/9402937/7dd0652a-480c-11e5-9138-bd14244593d5.png)
6. 提交 [pull request](https://help.github.com/articles/using-pull-requests/)！

## 更新你的 Pull Request

有时，awesome 列表的维护者会要求你在合并之前编辑 Pull Request。这通常是由于拼写错误或你的 PR 不符合 awesome-* 列表指南。

[这里](https://github.com/RichardLitt/knowledge/blob/master/github/amending-a-commit-guide.md) 是关于如何更改 Pull Request 以及你可以通过不同方式实现的文章。
