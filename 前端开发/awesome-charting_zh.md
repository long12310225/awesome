## 商业图书馆
* [amCharts](https://www.amcharts.com/) - 三个传统图表、库存和地图库。具有手绘风格主题选项。
* [AnyChart](http://www.anychart.com/) - 用于绘制不同类型数据图表的产品集。具有特殊的 Oracle Apex 集成选项。
* [CanvasJS](http://canvasjs.com/) - 库中的一组紧密的图表类型，文件大小较小。与其他商业库不同的默认主题。
* [FusionCharts](http://www.fusioncharts.com/) - 具有强大地图组件的图表库。
* [Highcharts](http://www.highcharts.com/) - 一系列用于多种用途的图表库。可以兼容回IE6。
* [JSCharting](https://JSCharting.com/) - 集成的图表库套件，包含传统图表类型以及地图、股票、甘特图、网格图和组织结构图。
* [Kendo UI](https://www.telerik.com/kendo-ui) - 高级图表库，具有可用于 Angular、React 和 Vue 的本机 UI 组件。
* [ZingChart](http://www.zingchart.com) - 旨在渲染其他库无法处理的大型数据集。超级可定制的样式和经常添加的新功能。一切都在一个模块化库中。

## 免费开源库
* [apexcharts.js](https://github.com/apexcharts/apexcharts.js) - 具有简单 API 的 JavaScript 图表库。
* [Billboard.js](https://naver.github.io/billboard.js/) - C3 的一个分支，可以更密切地跟踪 D3 的发布。
* [C3.js](http://c3js.org/) - 基于 D3 的可重用图表库。
* [Chart.js](http://www.chartjs.org/) - 小型库（11kb！），包括核心图表类型。
* [Charts.css](https://chartscss.org/) - CSS数据可视化框架。
* [Chartist](https://gionkunz.github.io/chartist-js/) - 简单、轻量级的图表库，使用 SVG 渲染图表，并使用 CSS 设计样式。
* [Chartkick](https://github.com/ankane/chartkick) - 带有一行 Ruby 的 JavaScript 图表。
* [D3.js](https://d3js.org/) - 允许用户根据数据操作文档以呈现 SVG 中的图表。
* [dc.js](https://dc-js.github.io/dc.js/) - 支持 [crossfilter](http://square.github.io/crossfilter/) 的 D3 库
* [dygraphs](https://github.com/danvk/dygraphs) - 交互式可缩放时间序列图表。
* [ECharts](https://github.com/ecomfe/echarts) - 一个强大的浏览器图表和可视化库。
* [EJSChart](https://github.com/EmpriseCorporation/EJSCharts) - 企业就绪图表库。
* [Graphosaurus](https://github.com/frewsxcv/graphosaurus) - 由 WebGL (三.js) 提供支持的 3D 图形查看器
* [lightweight-charts](https://github.com/tradingview/lightweight-charts) - 使用 HTML5 canvas 构建的金融轻量级图表
* [Morris.js](http://morrisjs.github.io/morris.js) - 用于渲染折线图、条形图、面积图和圆环图的简单 API
* [Plotly](https://github.com/plotly/plotly.js) - 构建在 d3 和 stack.gl 之上，允许用户创建基本图表和 SVG 地图。
* [Plottable](https://github.com/palantir/plottable) - 使用 OOP 风格语法来构建图表的库。
* [QuickChart](https://github.com/typpo/quickchart) - 呈现静态图表图像的 Web API。
* [rgraph](http://www.rgraph.net/) - 具有 google 表格导入功能的 2D/3D javascript 图表。
* [sigma.js](https://github.com/jacomyal/sigma.js) - 用画布构建的图形/网络图库。
* [Smoothie Charts](https://github.com/joewalnes/smoothie) - 用于实时流数据的 JavaScript 图表。
* [TauCharts](https://www.taucharts.com/) - 独特的语法使开发人员可以使用 DSL 描述数据。具有创造面的能力。
* [μPlot](https://github.com/leeoniya/uPlot) - 一个小型（< 25 KB 分钟）的快速图表，用于时间序列、折线、面积、ohlc 和条形图。
* [uvCharts](https://github.com/imaginea/uvCharts) - 使用 d3.js 构建的 JavaScript 图表库
* [vis.js](http://visjs.org/) - 网络图、带有标签的描述性时间线以及 3D 图表。
* [reaviz](https://reaviz.io) - React 的模块化图表组件库。
* [reaflow](https://reaflow.dev) - 用于构建静态或交互式编辑器的模块化图表引擎。

## 免费图书馆
* [Google Charts](https://developers.google.com/chart/) - 地图是拨入的（当然）和一些很酷的动画选项。广泛的文档和使用社区（论坛）。

## 特定于框架的库
### 角
* [Angular-Chart](http://jtblin.github.io/angular-chart.js) - 用于渲染折线图、条形图、面积图和圆环图的简单 API
* [n3-charts](https://github.com/n3-charts/line-chart) - 使用 AngularJS 编写的易于使用的库，使用 D3 渲染图表。
* [Angular-scale](https://github.com/kirillstepkin/scale) - 用于创建折线图的简单轻量级库

### 余烬
* [Ember Charts](http://addepar.github.io/ember-charts/#/overview) - 五种基本图表类型可供 Ember.js 项目使用。

### jQuery
* [Flot](http://www.flotcharts.org/) - jQuery 的 JavaScript 图表库。包含有用的图表类型，但看起来可能不再处于积极开发状态。
* [jqPlot](http://www.jqplot.com) - 用于绘制图表的开源 jQuery 插件。包含许多常用功能，但可能对这些项目使用不同的命名约定。
* [jqxChart](http://www.jqwidgets.com/jquery-widgets-documentation/documentation/jqxchart/jquery-chart-getting-started.htm) - 以 SVG、Canvas 和 VML 呈现大量图表类型。
* [JQuery-linechart](https://github.com/kirillstepkin/jquery-linechart) - 用于创建折线图的简单轻量级库

### Lit-html
* [lit-line](https://github.com/apinet/lit-line) - {快速、小型、交互式、完全响应式}折线图 Web 组件构建在 lit-html 之上。

### 反应
* [nivo](https://github.com/plouc/nivo) - React 组件可轻松构建基于 D3 的数据可视化应用程序。
* [react-vis](https://github.com/uber/react-vis) - Uber 的可组合可视化系统
* [vx](https://vx-demo.now.sh/) 可重用的低级可视化组件集合，位于 d3 之上
* [Potion](http://numberpicture.com/build) - 用于使用 React 和 D3 构建动画可视化的低级构建块。
* [react-chart-js](https://github.com/jerairrest/react-chartjs-2) - Chart.js 的 React 包装器
* [react-d3](https://github.com/esbullington/react-d3) - 依赖 React 生成 SVG 标记和 d3 计算路径值的图表库。
* [react-muze](https://github.com/chartshq/react-muze) - [muze](https://muzejs.org/) 的 React 包装器（免费数据可视化库，用于使用 WebAssembly 在浏览器中创建探索性数据可视化）
* [react-vis](https://github.com/uber-common/react-vis) - 用于渲染常见数据可视化图表的 React 组件集合
* [recharts](http://recharts.org) - 使用 React 和 D3 构建的重新定义的图表库
* [Victory](https://github.com/FormidableLabs/victory) - 用于构建交互式数据可视化的可组合 React 组件的集合

## 数据可视化资源
* [Dataviz Catalogue](http://datavizcatalogue.com) - 当您想知道哪种图表类型最好时，可以去这里。塞韦里诺·里贝卡的一个项目。
* [Evergreen Data](http://stephanieevergreen.com) - 有关更有效地呈现数据的信息和意见。
* [Flowing Data](http://flowingdata.com) - 由《纽约时报》前雇员 Nathan Yau 提供的有关地图、可视化、统计数据和设计的资源。
* [Junk Charts](http://junkcharts.typepad.com) - Dataviz 评论家（兼哥伦比亚大学教授）Kaiser Fung 分解了数据可视化，以找出问题所在，并提出改进方法。
* [The Functional Art](http://www.thefunctionalart.com) - 著名数据可视化作者兼教授 Aberto Cairo 的关于数据可视化趋势和最佳实践的博客。
* [Visualizing Data](http://www.visualisingdata.com) - Dataviz 顾问 Andy Kirk 汇编了有关图表和可视化主题的示例、资源和参考材料。
* [Well Formed Data](http://well-formed-data.net/archives/1210/little-boxes) - 世界经济论坛前任成员莫里茨·斯特凡纳 (Moritz Stefaner) 关于信息可视化和设计的思考文章。
