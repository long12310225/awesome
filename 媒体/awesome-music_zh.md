## 音频编辑

* [Audacity](https://github.com/audacity/audacity) - 免费的跨平台数字音频编辑器。
* [Ardour](http://ardour.org/) - 强调音频录制的跨平台数字音频工作站。
* [LMMS](https://lmms.io/) - 另一个跨平台数字音频工作站，更注重制作节拍。
* [OpenUtau](https://github.com/stakira/openutau/) - 一个免费的、跨平台的歌声合成框架。
* [snd](https://sourceforge.net/projects/snd/) - Snd 是一个大致模仿 Emacs 的声音编辑器。
* [Stargate DAW](https://github.com/stargatedaw/stargate/) - 一款跨平台、一体式数字音频工作站和插件套件，针对 EDM 制作进行了优化。
* [TuneFlow](https://tuneflow.com/) - 下一代 DAW，可帮助您生成旋律、节拍或整首歌曲，同时仍然能够对其进行编辑。

## 音频库

* [audio.js](https://github.com/kolber/audiojs) - audio.js 是一个嵌入式 JavaScript 库，允许 HTML5 的 <audio> 功能。标签可在任何地方使用。
* [audioFlux](https://github.com/libAudioFlux/audioFlux) - 用于音频和音乐分析、特征提取的库。
* [chord-mark](https://github.com/no-chris/chord-mark) - ChordMark 的参考实现，一种歌词、和弦和节奏的文本符号格式。
* [howler.js](https://github.com/goldfire/howler.js) - 用于现代网络的 Javascript 音频库。
* [JSyn](http://www.softsynth.com/jsyn/) - Java 音频合成软件 API。
* [Librosa](https://github.com/librosa/librosa) - 用于音乐和音频分析的 python 包，提供创建音乐信息检索系统所需的构建块。
* [midi.js](https://github.com/mudcube/MIDI.js) - 用于创建基于 Web 的 MIDI 应用程序的框架。
* [mxml](https://github.com/venturemedia/mxml) - MusicXML 解析和布局库。
* [PortAudio](http://www.portaudio.com/) - 一个免费、跨平台、开源的音频 I/O 库。
* [RustAudio](https://github.com/RustAudio) - 用 Rust 编写的免费且有用的音频、DSP 和音乐库的集合。
* [Soundpipe](https://pbat.ch/proj/soundpipe.html) - 用 C 语言编写的轻量级 DSP 库。

## 音频工具

* [Auralytics](https://github.com/WengYiNing/Auralytics) - 开源个人 Spotify 分析工具。
* [Beets](http://beets.io/) - 一个强大的命令行音乐组织器和操纵器。
* [Cecilia](https://github.com/belangeo/cecilia5) - 用于音乐和信号处理的基于 Pyo 的图形环境。
* [cyanrip](https://github.com/atomnuker/cyanrip) - 只需用户最少的努力即可翻录和编码标准音频 CD。跨平台。
* [fre:ac](https://github.com/enzo1982/freac) - 免费音频转换器。它支持音频 CD 翻录和标签编辑。
* [Jack](https://github.com/jack-cli-cd-ripper/jack) - 命令行 CD 抓取工具。
* [K3b](https://github.com/KDE/k3b) - 是一款功能齐全的 CD/DVD/蓝光刻录和翻录应用程序。
* [Kima](https://github.com/Chevron7Locked/kima-hub) - 一个自托管的音乐流服务器，具有 ML 支持的音频分析、基于振动的发现和 Subsonic API 支持。
* [Koel](https://github.com/phanan/koel) - 开源个人音频流服务器。
* [Lame](http://lame.sourceforge.net/) - 一个高质量、免费、开源的 MP3 编码器。
* [lltag](https://github.com/bgoglin/lltag) - 自动命令行 mp3/ogg/flac 文件标记器和重命名器。
* [matchering](https://github.com/sergree/matchering) - 用于自动参考音频母带制作的容器化 Web 应用程序和库。
* [MusicBrainz](https://musicbrainz.org) - 开放音乐信息和元数据百科全书，为所有内容提供唯一的 ID
* [Rubyripper](https://github.com/bleskodev/rubyripper) - 是用于类 Unix 操作系统的安全数字音频提取应用程序（“cd ripper”）。
* [SoX](http://sox.sourceforge.net/) - 一个免费的开源命令行实用程序，用于播放、转换和操作音频文件。
* [split2flac](https://github.com/ftrvxmtrx/split2flac) - 将 flac/ape/wv/wav + 提示表分割成单独的轨道
* [Whipper](https://github.com/JoeLametta/whipper) - Python CD-DA 抓取工具更看重准确性而不是速度。
* [X Audio Copy](https://github.com/giorgiofranceschi/xaudiocopy) - GTK 和 GNOME 应用程序用于翻录 CD 音频并以有损和无损音频格式进行编码。

## 免费乐谱

* [bayanguru](https://bayanguru.org/) - 雕刻和出版系统，可生成 PDF 格式的高质量巴扬乐谱。
* [cellist/Lilypond-Sheet-Music](https://github.com/cellist/Lilypond-Sheet-Music) - 用 lilypond 排版的音乐集（主要是室内乐和大提琴音乐）。
* [ciconia/music](https://github.com/ciconia/music) - 主要是巴洛克音乐的集合，J.S.巴赫的许多神圣作品，用 lilypond/ripple 编码。
* [CPDL](http://www.cpdl.org/) - 合唱公共领域库包含超过 20K 的免费声乐乐谱。
* [Éditions Nicolas Sceaux](https://github.com/nsceaux/nenuvar) - 巴洛克乐谱（主要是法国音乐），使用百合花排版。
* [IMSLP](http://imslp.org/) - 国际乐谱图书馆项目是网络上最大的公共领域乐谱和乐谱集合。
* [Lavender Blue Open-Scores](https://github.com/madrisan/open-scores) - 使用 lilypond 编码的开放钢琴乐谱。
* [Mutopia](https://github.com/MutopiaProject/MutopiaProject) - 基于公共领域版本的乐谱。
* [wbsoft/lilymusic](https://github.com/wbsoft/lilymusic) - 美丽的 LilyPond 在免费许可下得分。

## MIDI 工具和库

* [Gingoduino](https://github.com/sauloverissimo/gingoduino) - 用于嵌入式系统的音乐理论引擎，支持 MIDI 1.0/2.0 UMP。
* [JJazzLab](https://github.com/jjazzboss/JJazzLab) - 一个完整的基于 Midi 的自动背景音乐生成框架。
* [Midifile](http://midifile.sapp.org/) - 用于解析标准 MIDI 文件的 C++ 库。
* [Midimonster](https://github.com/cbdevnet/midimonster) - 适用于大多数显示控制协议的通用控制和转换工具。
* [mido](https://github.com/mido/mido) - 用于处理 MIDI 消息和端口的 Python 库。
* [mididings](https://github.com/mididings/mididings) - 适用于 Linux 和 macOS 的基于 Python 的 MIDI 路由器和处理器
* [Polyrhythmix](http://github.com/dredozubov/polyrhythmix) - MIDI 鼓
发生器，专为多节奏部分而设计
* [stagedings](https://github.com/mididings/stagedings) - 用于 mididings 的 UI 和 API
* [Timidity](http://timidity.sourceforge.net/) - 一个播放 MIDI 文件的开源命令行合成器。

## 乐谱

* [Abjad](http://abjad.mbrsi.org/) - 基于 Lilypond 的用于形式化分数控制的 Python API。
* [ChordMark](https://chordmark.netlify.app/) - 一种基于文本的歌词、和弦和节奏记谱格式。
* [Denemo](http://www.denemo.org/) - 基于 Lilypond 的免费乐谱编辑器。
* [Digital Music Stand](https://github.com/PatWie/digitalmusicstand) - 用于显示乐谱的免费网络应用程序。
* [Jan Angermüller's music fonts page](http://elbsound.studio/music_fonts.php) - 不同音乐字体的列表和视觉比较。
* [Frescobaldi](https://github.com/wbsoft/frescobaldi) - 免费的 Lilypond 乐谱编辑器。
* [Guido](http://guidolib.sourceforge.net/) - 用于乐谱图形渲染的通用、可移植库和 API。
* [Hacklily](https://www.hacklily.org) - 基于 LilyPond 的在线乐谱编辑器（带有可选的 GitHub 集成）。
* [Inknote](https://github.com/MichalPaszkiewicz/inknote) - 免费、开源、基于浏览器的乐谱和作曲软件。
* [JotChord](https://www.jotchord.com) - 一个基于文本的在线编辑器，用于制作 PDF 格式的纳什维尔数字系统歌曲图表。
* [LibMEI](https://github.com/rism-digital/verovio/tree/develop/libmei) - 用于读写 MEI 文件的 C++ 库。
* [Lilybin](http://lilybin.com/) - 一个基于 Web 的开源 LilyPond 编辑器。
* [Lilypond](http://lilypond.org/) - 免费、开源的音乐雕刻程序，致力于制作尽可能最高质量的乐谱。
* [Lilypond cook book](https://github.com/noteflakes/lilypond-cookbook/wiki/) - 充分利用 lilypond 的提示和技巧。
* [Lilypond Snippet Repository](http://lsr.di.unimi.it/) - Lilypond 片段想法、功能和技巧的在线数据库。
* [Lilyvm](https://github.com/olsonpm/lilyvm) - Lilypond 版本管理器。
* [Lyp](https://github.com/noteflakes/lyp) - Lilypond 瑞士军刀 - 轻松安装软件包、管理 Lilypond 的多个版本以及高级用户的其他工具。
* [mei-friend](https://mei-friend.mdw.ac.at) - 一个友好的、基于浏览器的音乐编码编辑器。
* [MuseScore](https://github.com/musescore/MuseScore) - 免费的开源乐谱和作曲软件。
* [MusicKit](https://github.com/venturemedia/musickit) - iOS 和 OSX 的乐谱渲染。
* [neoscore](https://neoscore.org/) - 用于以图形优先范例来标记音乐的 python 库。
* [NOTATIO](http://notat.io/) - 致力于乐谱练习的论坛。
* [OpenLilyLib](https://github.com/openlilylib/snippets/) - LilyPond 工具 - 片段、模板和扩展。
* [OpenSheetMusicDisplay](https://github.com/opensheetmusicdisplay/opensheetmusicdisplay) - 用于在浏览器中呈现 MusicXML 的 Typescript 库。
* [Python-ly](https://pypi.python.org/pypi/python-ly) - 用于操作 LilyPond 文件的工具和库。
* [Ripple](https://github.com/ciconia/ripple/) - DRY for Lilypond - 轻松生成乐谱和零件。
* [Scorelib](http://scorelib.sapp.org/) - 用于解析 SCORE 数据文件的 C++ 库。
* [Tbon](https://github.com/Michael-F-Ellis/tbon) - 音乐的印刷节拍符号。
* [Unison](https://unisonofficial.com/html/user-documentation.html?title=unison-editor) - 用文字创建乐谱的专有软件。
* [VectorScore](https://github.com/DrakeB1234/VectorScore) - 基于 SVG 的轻量级音乐记谱库，适用于 MIT 许可下的 TypeScript。
* [Verovio](https://github.com/rism-ch/verovio) - 用于将 MEI 音乐符号雕刻到 SVG 中的库和工具包。
* [Vexflow](https://github.com/0xfe/vexflow) - 用于渲染乐谱和吉他谱的 JavaScript 库。

## 音乐编程

* [Alda](https://github.com/alda-lang/alda) - 音乐家的音乐编程语言。
* [Bach: Automated Composer's Helper](http://www.bachproject.net/) - Max 的一组跨平台补丁和外部组件，旨在将丰富的计算机辅助合成带入实时世界。
* [Cane](https://github.com/Jackojc/cane) - 围绕矢量和欧几里德节奏设计的小型 MIDI 音序器 DSL。
* [Chuck](https://github.com/ccrma/chuck) - 强定时、并发和动态音乐编程语言。
* [CoffeeCollider](https://github.com/mohayonao/CoffeeCollider) - HTML5 中的实时音频合成和算法合成语言。 “编写CoffeeScript，并作为SuperCollider进行处理。”
* [Common Music](http://commonmusic.sourceforge.net/) - 一个开源音乐创作系统。
* [Csound](http://csound.github.io/) - 用户可编程和用户可扩展的声音处理语言和软件合成器。
* [DrumBeatRepo](https://github.com/babali42/drumbeatrepo) - 用于在浏览器中播放不同音乐流派的鼓点的节奏库。
* [Faust](http://faust.grame.fr/) - 专门为实时信号处理和合成而设计的函数式编程语言。
* [Gwion](https://github.com/Gwion/Gwion) - 一种旨在音乐创作的强定时编程语言。深受 ChucK 的启发。
* [IanniX](https://github.com/iannix/IanniX) - 一个基于 Iannis Xenakis 作品的图形开源音序器，用于数字艺术。
* [Inscore](http://inscore.sourceforge.net/) - 用于设计交互式增强乐谱的环境。
* [Leipzig](https://github.com/ctford/leipzig) - Overtone 的作曲库。
* [Leiter](https://github.com/kad1kad/leiter) - 基于 TypeScript 的库，旨在生成音阶。
* [Midica](https://github.com/truj/midica) - MIDI编程语言，带反编译器和播放器，支持卡拉OK。
* [music21](http://web.mit.edu/music21/) - 计算音乐学工具包。
* [Music Suite](http://music-suite.github.io/docs/ref/) - 一种基于 Haskell 的通用音乐符号和理论语言。
* [mutwo](https://mutwo-org.github.io/) - 一个用于合成的 Python 库，可输出乐谱、MIDI 或音频文件，并提供通用模型来描述基于时间的结构。
* [Nyquist](https://www.cs.cmu.edu/~music/nyquist/) - 健全的综合和作曲语言。
* [OpenMusic](http://repmus.ircam.fr/openmusic/home) - 一种基于 Lisp 的可视化编程语言。
* [Orca](https://github.com/hundredrabbits/Orca) - 音乐的现场编程环境。
* [Overtone](https://github.com/overtone/overtone/) - 用于设计合成器和与音乐协作的开源工具包。
* [Platonic Music Engine](http://www.platonicmusicengine.com/) - 用 Lua 编写的开源音乐生成框架。
* [Pure Data](http://puredata.info/) - 用于音频和其他多媒体的可视化编程语言。
* [pyo](https://belangeo.github.io/pyo/) - 帮助创建数字信号处理脚本的Python模块。
* [py-modular](http://py-modular.readthedocs.io/) - 一个模块化的实验性编程环境，带有 Python 中的基本 DSP 例程。
* [Sardine](https://github.com/Bubobubobubobubo/sardine) - Python 3.10+ 的音乐现场编码库（MIDI/OSC/SuperCollider）。
* [slippery chicken](https://github.com/mdedwards/slippery-chicken) - 一个 Common Lisp 环境，用于构建在 CLM、CMN、CM、MusicXML 和 Lilypond 上的作曲环境，用于乐谱、声音文件和/或 MIDI 文件输出。
* [Sonic Pi](http://sonic-pi.net/) - 强调教育用途的实时编码合成器。
* [Sporth](https://pbat.ch/proj/sporth.html) - 一种基于小型堆栈的音频语言。
* [Strudel](https://strudel.tidalcycles.org/) - 一项实验，旨在使用 Web 技术创建一种基于 javascript 的实时编码模式语言。
* [SuperCollider](http://supercollider.github.io/) - 一种用于实时音频合成和算法合成的编程语言。
* [Tidal](https://tidalcycles.org/) - 一种基于 Haskell 构建的用于实时编码模式的语言。
* [Topos](https://topos.live) - 基于网络的实时编码环境（WebAudio / MIDI）
* [tuneflow-py](https://github.com/tuneflow/tuneflow-py/) - 适用于下一代 DAW 的 python 插件 SDK，可让您将音乐算法和 AI 模型编程到音乐制作的所有阶段
* [Unilang](https://unisonofficial.com/html/user-documentation.html?title=unison-editor) - Unison 中使用的领域特定语言。

## 音乐理论教学

* [Artusi](https://www.artusi.xyz) - 互动音乐理论练习。
* [Four Score and More](https://fourscoreandmore.org/) - 根据乐谱量身定制的音乐理论练习。
* [musictheory.net](https://www.musictheory.net) - 课程和练习
* [OpenMusicTheory](http://openmusictheory.com/) - 一本不断发展的在线音乐理论和听觉技能“教科书”。
* [Theorytab](https://www.hooktheory.com/theorytab) - 带有和弦功能的歌曲数据库。

## 标准

* [MEI (Music Encoding Initiative)](http://music-encoding.org/) - 一项开源项目，旨在定义一种以机器可读结构对音乐文档进行编码的系统。
* [MusicXML](http://www.musicxml.com/) - 用于交换数字乐谱的标准开放格式。
* [SMuFL (Standard Music Font Layout)](http://www.smufl.org/) - 将音乐符号映射到 Unicode 以用于音乐字体的规范。
