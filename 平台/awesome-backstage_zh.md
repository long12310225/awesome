## 插件

一系列很棒的后台插件，可增强您平台的功能。

### 核心插件

- [Catalog](https://github.com/backstage/backstage/tree/master/plugins/catalog) - 提供生态系统中所有组件的清单。
- [TechDocs](https://github.com/backstage/backstage/tree/master/plugins/techdocs) - 管理和显示组件的技术文档。
- [Scaffolder](https://github.com/backstage/backstage/tree/master/plugins/scaffolder) - 使用模板创建新组件。
- [Cost Insights](https://github.com/backstage/community-plugins/tree/main/workspaces/cost-insights/plugins/cost-insights) - 深入了解您的云成本。

### 社区插件

- [Community Plugins](https://github.com/backstage/community-plugins) - Backstage 维护可用的社区插件列表。
- [GitHub Actions](https://roadie.io/backstage/plugins/github-actions/) - 监控您的 GitHub Actions 工作流程。
- [Jenkins](https://roadie.io/backstage/plugins/jenkins/) - 集成 Jenkins 以进行 CI/CD。
- [SonarQube](https://roadie.io/backstage/plugins/sonarqube/) - 使用 SonarQube 分析和监控代码质量。
- [CircleCI](https://roadie.io/backstage/plugins/circle-ci/) - 监控您的 CircleCI 构建。
- [Prometheus](https://roadie.io/backstage/plugins/prometheus/) - 使用 Prometheus 监控应用程序性能。
- [Bazaar](https://github.com/backstage/community-plugins/tree/main/workspaces/bazaar/plugins/bazaar) - 发现组织内的内部项目并进行协作。
- [backstage-aws-cloudformation-plugin](https://github.com/purple-technology/backstage-aws-cloudformation-plugin) - 管理 AWS CloudFormation 堆栈。
- [aws-lambda](https://roadie.io/backstage/plugins/aws-lambda/) - 管理和监控您的 AWS Lambda 函数。
- [AiKA](https://backstage.spotify.com/discover/blog/aika-data-plugins-coming-to-portal/) - 由 AI 驱动的助手，可从您的内部文档和 API 中获取上下文感知答案。
- [RAG AI Plugin](https://www.opslevel.com/resources/top-10-backstage-plugins-for-2025) - 使用法学硕士为您的内部知识添加检索增强生成。
- [Announcements Plugin](https://www.npmjs.com/package/@backstage-community/plugin-announcements) - 直接在后台创建和管理内部公告。
- [Git Release Manager](https://backstage.spotify.com/partners/spotify/plugin/git-release-manager/) - 在 Backstage 中自动化 GitHub 发布工作流程。
- [Port Plugin](https://www.port.io/blog/top-5-backstage-plugins) - 将整个堆栈中的数据同步到记分卡和仪表板中。
- [Env0 Plugin](https://www.env0.com/blog/new-backstage-plugin-manage-and-deploy-iac-from-your-internal-developer-portal) - 管理基础设施即代码 (IaC) 工作流程和自助服务配置。
- [Tech Insights Scorecards](https://roadie.io/backstage/plugins/tech-insights/) - 通过对整个目录进行集中检查，自动化服务成熟度记分卡。
- [Scaffolder Yeoman Module](https://www.npmjs.com/package/@backstage/plugin-scaffolder-backend-module-yeoman) - 使用 Yeoman 蓝图和新模板操作（例如 GitHub Issues 或 Terraform Cloud）扩展黄金路径。
- [Microsoft Graph Catalog Module](https://www.npmjs.com/package/@backstage/plugin-catalog-backend-module-msgraph) - 将 Azure AD 用户和组同步为 Backstage 实体。
- [Azure DevOps Catalog Modules](https://www.npmjs.com/package/@backstage/plugin-catalog-backend-module-azure) - 将 Azure Repos、Boards 和 Pipelines 元数据引入目录后端。
- [Search Backend Modules](https://www.npmjs.com/package/@backstage/plugin-search-backend-module-elasticsearch) - 使用 Elasticsearch 或 Lunr 支持的混合索引解锁更快的过滤搜索。
- [New Relic](https://www.npmjs.com/package/@backstage-community/plugin-newrelic) - 从服务目录条目直接跳转到 New Relic 仪表板。
- [Apiiro](https://github.com/backstage/community-plugins/blob/main/workspaces/apiiro/plugins/apiiro/README.md) - 直接在 Backstage 实体中显示 Apiiro 应用程序风险上下文。
- [Wiz Security](https://github.com/wiz-sec-public/backstage-plugin-wiz) - 在记分卡旁边显示 Wiz 云风险评分，以确定修复的优先顺序。
