## 外观

网站的外观或可见方面。

+ **[动画](appearance/animation.md)**：创建运动和形状变化的过程。
+ **[Typography](appearance/typography.md)**：排版内容的样式、排列或外观。
+ **[可视化](appearance/visualization.md)**：将数据放置在视觉上下文中。

## 建筑

前端代码的高级结构以及创建此类结构的规则。

+ **[算法](architecture/algorithms.md)**：要执行的独立的分步操作集。算法执行计算、数据处理和/或自动推理任务。
+ **[设计模式](architecture/design-patterns.md)**：程序员在设计应用程序或系统时可以用来解决常见问题的最佳实践。
+ **[Designs](architecture/designs.md)**：用于前端开发的随时可用且记录良好的结构和框架。
+ **[事件驱动编程](architecture/event-driven-programming.md)**：事件驱动编程是一种编程范例，其中程序的流程由事件决定，例如用户操作、传感器输出或来自其他程序/线程的消息。
+ **[函数式编程](architecture/function-programming.md)**：函数式编程是一种编程范式，它将计算视为数学函数的评估，并避免改变状态和可变数据。
+ **[函数响应式编程 (FRP)](architecture/function-reactive-programming-frp.md)**：FRP 是一种使用函数式编程构建块进行异步数据流编程的编程范例。

## 兼容性

产品与不同输入/输出设备和渲染软件配合使用的能力。包括打印机、电子邮件、移动设备和不同的浏览器。

+ **[跨浏览器](compatibility/cross-browser.md)**：跨浏览器是指网站、Web 应用程序、HTML 构造或客户端脚本在提供其所需功能的环境中运行的能力，以及在功能不存在或缺乏时优雅退出或降级的能力。
+ **[E-Mail](compatibility/e-mail.md)**：准备基于 HTML 的电子邮件。
+ **[键盘](compatibility/keyboard.md)**：在 Web 浏览器中使用键盘输入。
+ **[Mobile](compatibility/mobile.md)**：开发针对在智能手机和平板电脑设备上查看而优化的网站。
+ **[打印机](compatibility/printers.md)**：通过 CSS 操作打印机输出。
+ **[响应式网页设计 (RWD)](compatibility/responsive-web-design-rwd.md)**：RWD 响应用户及其所使用设备的需求。布局根据设备的尺寸和功能而变化。
+ **[Web 可访问性](compatibility/web-accessibility.md)**：Web 可访问性意味着残疾人可以感知、理解、导航并与 Web 交互，并且可以为 Web 做出贡献。

## 语言、协议、浏览器 API

编程/标记语言和网络相关标准。

+ **[级联样式表 (CSS)](languages-protocols-browser-apis/cascading-style-sheets-css.md)**：CSS 是一种样式表语言，用于描述以 HTML 或 XML 编写的文档的表示形式。它描述了如何在屏幕、纸张、语音或其他媒体上呈现元素。
+ **[文档对象模型 (DOM)](languages-protocols-browser-apis/document-object-model-dom.md)**：DOM 是 HTML、XML 和 SVG 文档的编程接口。它定义了允许访问树的方法，以便它们可以更改文档结构、样式和内容。
+ **[超文本标记语言 (HTML)](languages-protocols-browser-apis/hypertext-markup-language-html.md)**：HTML 是用于创建网页的标准标记语言，其元素构成了所有网站的构建块。
+ **[超文本传输​​协议 (HTTP)](languages-protocols-browser-apis/hypertext-transfer-protocol-http.md)**：超文本传输​​协议 (HTTP) 是一种用于分布式、协作式超媒体信息系统的应用程序协议。 HTTP 是万维网数据通信的基础。
+ **[JavaScript (EcmaScript)](languages-protocols-browser-apis/javascript-ecmascript.md)**：JavaScript 是一种成熟的动态编程语言，当应用于 HTML 文档时，可以在网站上提供动态交互性。它是由 ECMAScript 标准定义的。
+ **[JavaScript 对象表示法 (JSON)](languages-protocols-browser-apis/javascript-object-notation-json.md)**：JSON（JavaScript 对象表示法）是一种轻量级数据交换格式。人类很容易阅读和书写。机器很容易解析和生成。它基于 JavaScript 编程语言的子集。
+ **[可缩放矢量图形 (SVG)](languages-protocols-browser-apis/scalable-vector-graphics-svg.md)**：一种基于 XML 的矢量图像格式，用于二维图形，支持交互性和动画。
+ **[Service Workers](languages-protocols-browser-apis/service-workers.md)**：一种使应用程序能够利用持久后台处理的方法，包括在离线时启用 Web 应用程序引导的挂钩。
+ **[模板语言和引擎](languages-protocols-browser-apis/templated-languages-and-engines.md)**：模板引擎是将程序逻辑和表示分离为两个独立部分的工具。这使得逻辑和表示的开发变得更加容易，提高了灵活性并简化了修改和维护。
+ **[Transpiled Languages](languages-protocols-browser-apis/transpiled-languages.md)**：抽象语言转换为原生的、浏览器支持的标准，如 JavaScript 或 CSS。
+ **[统一资源标识符 (URI)](languages-protocols-browser-apis/uniform-resource-identifier-uri.md)**：URI 是用于标识资源的字符串。 URI 最常见的形式是统一资源定位符 (URL)。
+ **[Web 动画 API](animation/web-animations-api.md)**：Web 动画是一种新的 JavaScript API，用于驱动网络上的动画内容。通过统一 SVG 和 CSS 的动画功能，Web Animations 解锁了以前只能以声明方式使用的功能，并向开发人员展示了强大的高性能动画功能。
+ **[WebAssembly](languages-protocols-browser-apis/web assembly.md)**：WebAssembly 旨在填补 JavaScript 迄今为止被迫占据的位置：可以用作编译器目标的低级代码表示形式。

## 用户界面组件

用于网站和应用程序的嵌入式 UI 组件。

+ **[按钮](user-interface-components/buttons.md)**：术语“按钮”指的是任何图形控制元素，它为用户提供了一种简单的方式来触发事件（例如在搜索引擎中搜索查询）或与对话框交互（例如确认操作）。
+ **[Code](user-interface-components/code.md)**：专为嵌入网站而设计的代码查看器和编辑器。
+ **[表单](user-interface-components/forms.md)**：网页上的 HTML 表单允许用户输入发送到服务器进行处理的数据。 Web 用户使用复选框、单选按钮或文本字段填写表单。
+ **[画廊和图像滑块](user-interface-components/galeries-and-image-sliders.md)**：一种在网站上呈现图像集合的复杂方式。
+ **[Grid](user-interface-components/grid.md)**：CSS 网格布局系统。
+ **[富文本编辑器](user-interface-components/rich-text-editors.md)**：富文本编辑器是在 Web 浏览器中编辑富文本的界面。目的是减少用户尝试将其格式直接表达为有效 HTML 标记的工作量。
+ **[目录](user-interface-components/table-of-contents.md)**：自动生成目录的组件。
+ **[UI Kits](user-interface-components/ui-kits.md)**：即用型组件的集合。
+ **[视频和音频](user-interface-components/video-and-audio.md)**：用于在网站上播放音频和视频文件的组件。

## 工作流程

任务自动化和资产交付。

+ **[自动化测试](workflow/automated-testing.md)**：自动化软件测试是软件工具在将软件应用程序发布到生产环境之前对其执行预先编写的测试的过程。
+ **[构建工具](workflow/build-tools.md)**：工具包及其生态系统，可帮助您自动执行痛苦且重复的任务。
+ **[CSS 工具](workflow/css-tools.md)**：用于分析、CSS 文件预处理和后处理的工具。
+ **[代码编辑器](workflow/code-editors.md)**：专为编辑网站源代码而设计的文本编辑器程序。
+ **[文档](workflow/documentation.md)**：编写、生成、发布和使用 Web 可交付成果的文档。
+ **[程序员的字体](workflow/fonts-for-programmers.md)**：程序员需要特殊的字体，这有助于对齐代码并区分看起来相似的字符。
+ **[入门](workflow/getting-started.md)**：设置前端开发工作流程的分步指南。
+ **[HTML 工具](workflow/html-tools.md)**：用于 HTML 源代码的预处理和后处理的工具。
+ **[图像后处理](workflow/image-post-processing.md)**：图像转换和优化工具。
+ **[JavaScript 工具](workflow/javascript-tools.md)**：用于 JavaScript 文件静态分析、预处理和后处理的工具。
+ **[包管理](workflow/package-management.md)**：包管理器或包管理系统是软件工具的集合，它以一致的方式自动执行安装、升级、配置和删除可重用库和组件的过程。
+ **[Sourcemaps](workflow/sourcemaps.md)**：Sourcemap 是一种将组合/缩小文件映射回未构建状态的方法。
+ **[版本控制](workflow/version-control.md)**：版本控制或源代码控制是记录一个文件或一组文件随时间变化的系统，以便您以后可以调用特定版本。
