这是一个协作社区策划的列表，其中包含很棒的 Ansible 资源、工具、角色、教程和其他相关内容。

[Ansible](https://www.ansible.com/) 是一个开源工具包，用 Python 编写，用于配置管理、应用程序部署、持续交付、IT 基础设施自动化和一般自动化。

：警告：在编辑列表以添加项目之前，请查看 [Ansible 生态系统项目开发资源](https://docs.ansible.com/ansible/devel/community/ecosystem_project_resources.html)。让我们确保我们的用户、贡献者和维护者拥有最好的体验！

## 工具

> 用于和使用 Ansible 的工具。

- [Automation Controller](https://www.ansible.com/products/controller) - 这款红帽产品以前称为 Ansible Tower，可帮助您扩展 IT 自动化、管理复杂的部署并提高生产力。将 Ansible 的力量扩展到您的整个团队。
- [AWX](https://github.com/ansible/awx) - 基于 Web 的用户界面、REST API 和构建在 Ansible 之上的任务引擎。它是自动化控制器的上游项目。
- [Ansible Lint](https://github.com/ansible/ansible-lint) - 检查 Playbook 中是否有可能改进的最佳实践和行为。
- [Ansible Doctor](https://github.com/thegeeklab/ansible-doctor) - 简单的注释，例如基于 Jinja2 模板的 Ansible 角色的文档生成器。
- [Ansible DocSmith](https://github.com/foundata/ansible-docsmith) - 从自述文件和默认变量文件的 argument_specs.yml 生成 Ansible 角色文档。
- [Ansible cmdb](https://github.com/fboender/ansible-cmdb) - 获取 Ansible 事实收集的输出并将其转换为静态 HTML 页面。
- [ARA](https://github.com/ansible-community/ara) - 记录 Ansible 剧本，并通过报告 API、UI 和 CLI 使其更易于理解和故障排除。
- [Ansible Inventory Grapher](https://github.com/willthames/ansible-inventory-grapher) - 显示清单继承层次结构以及在清单中定义变量的级别。
- [Mitogen for Ansible](https://mitogen.networkgenomics.com/ansible_detailed.html) - 使用 Mitogen 大幅加快 Ansible 速度。
- [Molecule](https://molecule.readthedocs.io/en/latest/) - 用于开发和测试 Ansible 角色的框架。
- [Packer Ansible Provisioner](https://www.packer.io/plugins/provisioners/ansible/ansible-local) - 此 Provisioner 可用于通过 Packer 和 Ansible 自动创建 VM 映像。
- [Excel Ansible Inventory](https://github.com/KeyboardInterrupt/ansible_xlsx_inventory) - 将任何 Excel 电子表格转变为 Ansible 库存。
- [terraform.py](https://github.com/mantl/terraform.py) - 用于解析 Terraform 状态文件的 Ansible 动态清单脚本。
- [Ansible101](https://ansible101.com) - 交互式零设置沙箱，用于通过实时可视化执行流程构建和调试剧本。
- [ansible-navigator](https://github.com/ansible/ansible-navigator) - Ansible 基于文本的用户界面 (TUI)。
- [squest](https://hewlettpackard.github.io/squest/) - 自动化控制器作业模板的自助服务门户。
- [ansible-bender](https://ansible-community.github.io/ansible-bender/build/html/index.html) - 使用 Ansible playbook 弯曲容器并将其转换为容器映像的工具。
- [ansible-runner](https://github.com/ansible/ansible-runner) - 一种工具和 Python 库，可在直接与 Ansible 交互或作为另一个系统的一部分进行交互时提供帮助，无论是通过容器映像接口、作为独立工具还是作为可导入的 Python 模块。
- [ansible-builder](https://ansible-builder.readthedocs.io/en/latest/) - 使用依赖于非默认依赖项的 Ansible 内容可能会很棘手。软件包必须安装在每个节点上，与主机系统上安装的其他软件很好地配合，并保持同步。
- [kics](https://github.com/Checkmarx/kics) - SAST 工具可将您的 Ansible 基础设施作为代码手册进行扫描，以查找安全漏洞、合规性问题和错误配置。
- [ansible-security-scanner](https://github.com/cpeoples/ansible-security-scanner) - 针对剧本、角色和集合的静态分析器，涵盖硬编码凭证、RCE 和供应链风险。输出 SARIF、CycloneDX SBOM 和 GitLab SAST。
- [php-ansible Library](https://github.com/maschmann/php-ansible) - Ansible 的 OOP 包装器，使 Ansible 在 PHP 中可用。
- [TD4A](https://github.com/cidrblock/td4a) - 用于构建和测试 jinja2 模板的设计辅助工具，将 yaml 格式的数据与 jinja2 模板相结合并渲染输出。
- [Ansible Playbook Grapher](https://github.com/haidaraM/ansible-playbook-grapher) - 命令行工具，用于创建代表您的 Ansible 剧本、任务和角色的图表。
- [ansible-doc-extractor](https://github.com/xlab-steampunk/ansible-doc-extractor) - 一种从 Ansible 模块中以 HTML 形式提取文档的工具。
- [Ansible Semaphore](https://github.com/ansible-semaphore/semaphore) - 用于管理和运行 Ansible playbook 的现代 UI。
- [Steampunk Spotter](https://steampunk.si/spotter/) - 提供辅助自动化编写工具，可分析您的 Ansible Playbook 并提供建议。
- [ansible-roster](https://gitlab.com/jlecomte/ansible/ansible-roster) - Ansible Roster 库存插件，用于从面向主机的 yaml 文件生成库存。支持范围、正则表达式主机名、文件包含和变量合并。
- [Monkeyble](https://hewlettpackard.github.io/monkeyble/) - 一个回调插件，允许使用 Pythonic 测试和 CI/CD 方法在 playbook 上执行端到端测试来检测回归。
- [aar-doc - Automated Ansible Role Documentation](https://github.com/telekom-mms/Automated-Ansible-Role-Documentation) - 从 Ansible 角色的元数据自动生成文档。
- [antsichaut](https://github.com/ansible-community/antsichaut) - 自动填充 antsibull-changelog 使用的changelog.yaml。
- [ansibledb](https://github.com/nbentoumi/ansibledb) - Flask API Web 服务器使用 MongoDB 作为数据库来存储 Ansible 报告和事实；该工具可用于查询 Ansible 管理的主机和因素以及搜索 Ansible 日志。
- [Ansible Template Playground](https://tech-playground.com/playgrounds/ansible-template/) - 用于运行、测试和共享 Ansible 模板的在线游乐场。
- [YAML 验证器](https://yamlvalidator.dev)、[(chrome 扩展)](https://chromewebstore.google.com/detail/yaml-validator/gjgbohnlhijomhfiflapnlnmcpckgigg) - 在线 YAML 验证器和格式化程序，具有 Ansible JSON 架构验证功能。 Chrome 扩展还在 GitHub 上添加了 YAML 代码折叠。

- [Linuxfabrik LFOps](https://github.com/Linuxfabrik/lfops) - Ansible 集合，包含 145 多个剧本和 160 多个角色，用于引导和管理 Linux 基础设施（RHEL 8/9/10、Debian、Ubuntu）。涵盖操作系统强化、MariaDB、Icinga2、Nextcloud、FreeIPA、KVM 和 Bitwarden 集成。

## 编辑器和 IDE 集成

> 与文本编辑器和 IDE 的出色集成，使使用 Ansible 进行开发变得更加容易。

- [Ansible Language Server](https://github.com/ansible/ansible-language-server) - 语言服务器向兼容的编辑器添加了对 Ansible 的支持。
- [VS Code - official Ansible Extension](https://marketplace.visualstudio.com/items?itemName=redhat.ansible) - 通过利用 ansible-language-server 为 Visual Studio Code 和 OpenVSX 兼容编辑器添加对 Ansible 的语言支持。
<!--lint disable -->
- [Vim](https://www.vim.org/) - 一个开源、免费的命令行文本编辑器。有用的 Vim 插件包括：
  - [Ansible vim](https://github.com/pearofducks/ansible-vim) - Ansible 2.x 的 vim 语法插件，它支持 YAML playbook、Jinja2 模板和 Ansible 的主机文件。
  - [Ansible vim and neovim plugin](https://www.npmjs.com/package/@yaegassy/coc-ansible) - Ansible 的 vim 插件（lsp 客户端），它支持自动完成、语法突出显示、悬停、诊断和 goto 支持。
- [Emacs](https://www.gnu.org/software/emacs/) - 一个免费的开源文本编辑器和 IDE，支持自动缩进、语法突出显示和内置终端 shell（以及其他功能）：
  - [lsp-mode](https://emacs-lsp.github.io/lsp-mode/page/lsp-ansible/) - Emacs 的 Ansible 语言服务器协议支持：提供语法突出显示、自动完成和诊断。
  - [yaml-mode](https://github.com/yoshiki/yaml-mode) - YAML 突出显示和语法检查。
  - [jinja2-mode](https://github.com/paradoxxxzero/jinja2-mode) - Jinja2 突出显示和语法检查。
  - [magit-mode](https://github.com/magit/magit) - Emacs 中的 Git 瓷器。
  - [flymake-ansible-lint](https://github.com/jamescherti/flymake-ansible-lint.el) - Ansible Lint 与编辑时自动/连续注释错误、警告和信息集成。
- [PyCharm](https://www.jetbrains.com/pycharm/) - 用于 Python 软件开发的完整 IDE。有用的插件包括：
  - [Ansible Lint](https://plugins.jetbrains.com/plugin/20905-ansible-lint) - Ansible Lint 与编辑时自动/连续注释错误、警告和信息集成。
  - [Ansible Vault Integration](https://plugins.jetbrains.com/plugin/14353-ansible-vault-integration) - IntelliJ IDEA 的 Ansible 保管库集成，具有用于保管和取消保管机密的上下文操作。
