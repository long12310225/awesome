## 架构示例
一个好的 ARCHITECTURE.md 文件可以帮助开发人员了解如何以及在哪里进行更改，无论他们是否是项目的新手。

图表、源代码图以及讨论不变量和设计决策是必不可少的。

这也可以是 README.md 文件的专用部分。
- [esbuild](https://github.com/evanw/esbuild/blob/main/docs/architecture.md) - 充分利用图形进行可视化和项目结构。包括该项目的重要原则列表。
- [Flutter Engine](https://github.com/flutter/flutter/blob/master/docs/about/The-Engine-architecture.md) - 充分利用高级图表来显示堆栈及其组成部分。描述主要流程。描述平台不变量。
- [GitLab](https://gitlab.com/gitlab-org/charts/gitlab/-/tree/master/doc/architecture) - 提出设计决策。
- [Linux cryptography](https://github.com/torvalds/linux/blob/master/Documentation/crypto/architecture.rst) - 调出不同类型的组件，提供可搜索区域，调出不同组件的不变量，并用图表描述结构。
- [Neovim](https://github.com/neovim/neovim/blob/master/src/nvim/README.md) - 描述主要流程/生命周期。
- [Oh My Zsh](https://github.com/ohmyzsh/ohmyzsh/wiki/Design) - 描述初始化过程，指出环境要求。
- [Redis](https://github.com/redis/redis/blob/unstable/README.md) - 很好的源代码图。关键文件概述。在代码中充分利用文档注释而不是内联注释。
- [Tauri](https://github.com/tauri-apps/tauri/blob/dev/ARCHITECTURE.md) - 制作精良的源代码图，讨论了架构注意事项，指出了重要的依赖关系。
- [VS Code](https://github.com/microsoft/vscode/wiki/Source-Code-Organization) - 善于使用高级图表。描述源组织。

## 创建 GIF

在自述文件中嵌入动画 gif 可以快速演示您的项目的用途并吸引读者的眼球。以下是一些可以帮助您快速为项目创建 gif 的程序：

- [Gifski](https://github.com/sindresorhus/Gifski#readme) - **免费** - 比其他颜色更鲜艳，但仍然保持较小的尺寸。
- [Giphy Capture](https://giphy.com/apps/giphycapture) - **免费** - 易于上传到 giphy.com，但用户体验有点烦人。
- [LICEcap](https://www.cockos.com/licecap/) - **免费** - 不太直观，但具有更多功能。
- [Peek](https://github.com/phw/peek#readme) - **免费** - Linux 用户简单易用。
- [ScreenToGif](https://github.com/NickeManarin/ScreenToGif/) - **免费** - 开源，具有可定制的 UI 和易于编辑的 GIF，易于上手。
- [terminalizer](https://github.com/faressoft/terminalizer) - **免费** - 录制您的终端并生成动画 GIF 图像或共享网络播放器。
- [ttystudio](https://github.com/chjj/ttystudio#readme) - **免费** - 对于命令行工具，终端到 GIF 录制器消除了麻烦。
