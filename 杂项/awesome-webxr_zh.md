## 杰出项目

*令人印象深刻且著名的 WebXR 项目。*

- [Hello WebXR](https://msub2.github.io/hello-webxr/) - 为纪念 WebXR 1.0 API 的发布而制作，展示了 WebXR 的一些可能功能。
- 这链接到我自己的分叉上的体验镜像，因为它不再可以在其原始域中访问。
- [Moonrider](https://moonrider.xyz/) - Supermedium打造的一款类似于Beat Saber的节奏游戏。
- [Plockle](https://plockle.com/) - 一款超过 40 个关卡的空间方块益智游戏。
- [Castle Builder](https://castle.needle.tools/) - 一款协作式跨平台城堡建造游戏，利用独特的 Unity-to Three.js 管道。
- [Prehistoric Domain](https://www.prehistoricdomain.com/) - 在浏览器中体验侏罗纪公园式的体验，提供多个恐龙展览，并且未来还会有更多的规划。
- [Above Par-adowski WebXR Mini Golf](https://aboveparadowski.com/) - 由 Paradowski Creative 设计的 WebXR 迷你高尔夫游戏，拥有 9 个洞，分布在独特且多样化的环境中。
- [Yolopia](https://www.yolopia.com/) - 一款融合了流动、拳击和姿势匹配的节奏游戏。
- [f60](https://heyvr.io/game/f60) - 以苏联监狱为背景的跨平台 WebXR 密室逃脱系列。
- [The Escape Artist](https://esc.art/) - Paradowski Creative 推出的一款屡获殊荣的密室逃脱游戏，以艺术家的思维为背景，对创作过程进行了总体叙述。
- [NeoFables](https://neofables.com/) - 利用 GenAI 动态生成的 VR 讲故事体验。获得 2024 年 Polys Immersive Awards 的“年度游戏”和“年度体验”提名，是 Meta 推出的首个 GenAI VR 体验。
- [XRSH](https://xrsh.isvery.ninja) - 在 WebXR 内运行的 Linux 虚拟机可在 VR 外壳内启用实时脚本、包和原型设计。 [[repo]](https://codeberg.org/xrsh/xrsh)

## 艺术

*以艺术创作为中心的项目。*

- [Brushwork VR](https://brushworkvr.com/) - 一款 VR 应用程序，可使用虚拟画布、调色板、颜色混合等来模拟绘画过程。
- [Silk Brush](https://msub2.github.io/silk-brush/) - Tilt Brush 到 WebXR 的端口以及未来扩展的计划。
- [VARTISTE](https://vartiste.xyz/) - 用于 2D 绘图和图像编辑以及 3D 模型纹理的 VR 应用程序。

## 音频

*以音频和视听体验为中心的项目。*

- [Fathom VR](https://www.letsfathom.com/vr) - 音乐发现应用程序 Fathom 的 WebXR 版本，可让您通过空间化音频搜索和探索相关艺术家的云。
- [TRACK](https://demos.littleworkshop.fr/track) - 由 [Little Workshop](https://www.littleworkshop.fr/) 开发的沉浸式视听体验线性骑行。

## 头像

*用于在沉浸式体验中创建和利用 3D 化身的资源。*

- [Exokit Avatars](https://github.com/exokitxr/avatars) - 基于网络的头像系统，可以加载多种不同模型格式的人形头像，并具有手臂+腿 IK、通过视位进行面部动画等功能。原始存储库不再维护，但[此处](https://github.com/msub2/avatars)有一个更新的分支。
- [VRoid](https://vroid.com/en) - 可让您创建动漫风格的 3D 头像，并导出为高度兼容的 .vrm 格式。

## 发展

*一些使 WebXR 内容成为可能的工具。有关完整列表，请参阅单独的 WebXR 创建工具列表。有关更深入的列表，请参阅 [awesome-webxr-development](https://github.com/Pico-Developer/awesome-webxr-development) 列表。*

### 发动机

- [Godot](https://godotengine.org/) - 自版本 3.2.4 起支持 WebXR 的跨平台开源引擎。
  - [官方文档](https://docs.godotengine.org/en/stable/classes/class_webxrinterface.html)

- [Needle](https://needle.tools/) - Blender 和 Unity 的引擎和导出器。
  - [官方文档](https://engine.needle.tools/docs/)
 
- [Onirix Studio](https://www.onirix.com/) - 用于在浏览器中创建 Web AR 体验的无代码编辑器。如果目标设备/浏览器不兼容，它会模拟 WebXR 的功能。

- [PlayCanvas](https://playcanvas.com/) - 基于浏览器的 WebGL 引擎，支持 WebXR 内容、移动浏览器、协作编辑等。
  - [官方文档](https://developer.playcanvas.com/en/)

- [Rogue Engine](https://rogueengine.io/) - 基于 Three.js 和 TypeScript 的 HTML5 游戏引擎。
  - [官方文档](https://rogueengine.io/docs/manual/getting-started/installation/)
  
- [Unity](https://unity.com/) - 一个非常流行的游戏引擎，带有社区包，可以将内容导出到 WebXR。
  - [WebXR Export](https://github.com/De-Panther/unity-webxr-export) - 由 [De-Panther](https://github.com/De-Panther/) 导出
  - [SimpleWebXR](https://github.com/Rufus31415/Simple-WebXR-Unity) - 由 [Rufus31415](https://github.com/Rufus31415) 导出

- [Wonderland Engine](https://wonderlandengine.com/) - 专为构建 WebXR 应用程序而设计的开发平台。
  - [官方文档](https://wonderlandengine.com/documentation/)

### 框架和库

- [A-Frame](https://aframe.io/) - 允许您使用基于 Three.js 构建的声明性 HTML 和自定义 JavaScript 组件来创建 VR 体验。
  - [官方文档](https://aframe.io/docs/)

- [Babylon.js](https://www.babylonjs.com/) - 打包到 JavaScript 框架中的强大且功能丰富的渲染引擎。
  - [官方文档](https://doc.babylonjs.com/)

- [JanusWeb](https://github.com/jbaicoianu/janusweb) - JanusVR 的浏览器内实现，可让您构建社交 VR 体验。

- [JSAR-DOM](https://github.com/m-creativelab/jsar-runtime) WHATWG DOM、CSSOM 和 WebXR 的 TypeScript 重新实现，用于构建在 Babylon.js 之上的 XR 应用程序。

- [p5.xr](https://p5xr.org/) - p5.j​​s 库的附加组件，可让您创建身临其境的 p5 草图。

- [react-three-fiber](https://github.com/pmndrs/react-three-fiber) - Three.js 的 React 渲染器。 [React XR](https://github.com/pmndrs/react-xr) 中还提供了其他特定于 WebXR 的挂钩和组件。

- [reactylon](https://www.reactylon.com/) - 一个强大的多平台框架，构建在 Babylon.js 和 React 之上，旨在创建 3D/XR 体验。
  - [官方文档](https://www.reactylon.com/docs)

- [three.js](https://threejs.org/) - 用于使用 WebGL 渲染 3D 内容的 JavaScript 库，为许多现有的 WebXR 体验和工具提供支持。
  - [官方文档](https://threejs.org/docs/index.html#manual/en/introduction/Creating-a-scene)

- [Threlte](https://threlte.xyz/) - Three.js 的 Svelte 渲染器和组件库。 WebXR 特定的挂钩和组件可在 [`@threlte/xr` 包](https://threlte.xyz/docs/reference/xr/getting-started) 中找到。

- [X3DOM](https://x3dom.org) - 用于将 X3D 场景集成和操作为 HTML5/DOM 元素的框架。
  - [官方文档](https://doc.x3dom.org/)

### 其他

- [Verge3D](https://www.soft8soft.com/verge3d/) - 该工具包允许 3D 艺术家导出场景并轻松创建身临其境的 Web 体验。
  - [官方文档](https://www.soft8soft.com/support-documentation/)
- [XR Fragments](https://xrfragment.org) - 一个基于现有元数据使用 URL 控制任何 3D 模型的小型规范。使用所有 3D 编辑器和查看器促进超链接的 WebXR 故事讲述。 [[repo]](https://codeberg.org/coderofsalvation/xrfragment)

## 社交空间
