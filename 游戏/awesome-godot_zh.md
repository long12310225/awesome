## 项目

*使用 Godot 制作的非游戏开源项目（工具/实用程序）。*

#### 戈多4

- [Arrow](https://github.com/mhgolkar/Arrow) - 一种用节点设计游戏叙事的工具。
- [Clipboard Narrator](https://github.com/lesleyrs/clipboard-narrator) - 将任何网页变成有声读物，在桌面后台运行！
- [GodSVG](https://github.com/MewPurPur/GodSVG) - 创建优化的可缩放矢量图形文件的工具。
- [Lorien](https://github.com/mbrlabs/Lorien) - 适用于 Windows、Linux 和 macOS 的无限画布绘图/白板应用程序。支持绘图板和压力感应。
- [Material Maker](https://github.com/RodZill4/material-maker) - 按程序创建 PBR 材质（类似于 Substance Designer）。
- [Pixelorama](https://github.com/Orama-Interactive/Pixelorama) - 2D 像素艺术编辑器。
- [ProtonGraph](https://github.com/protongraph/protongraph) - 用于程序内容创建的基于节点的工具。与可视化脚本类似，但用于 3D 模型生成（需要自定义引擎模块）。

#### 戈多3

- [Godello](https://github.com/alfredbaudisch/Godello) - 用 Godot 制作的类似 Trello 的看板应用程序。使用 Godot 进行复杂的非游戏应用程序以及多个用户之间的实时在线交互的概念验证。包括后端代码。
- [SimulaVR](https://github.com/SimulaVR/Simula) - 适用于 Linux 的 VR 窗口管理器，构建于 Godot 引擎之上。
- [vpuppr](https://github.com/virtual-puppet-project/vpuppr) - 具有多个跟踪后端的跨平台 VTuber 应用程序。

## 模板

*用作您自己的游戏基础的项目。*

#### 戈多4

- [COGITO](https://codeberg.org/Phazorknight/Cogito) - 第一人称沉浸式模拟模板项目，提供了一个用于创建可交互对象、各种要使用的物品以及影响玩家和游戏环境的机制的框架。
- [Crystal Bit Godot Game Template](https://github.com/crystal-bit/godot-game-template) - 有意见的游戏模板。它包括持续集成、带有图形过渡的场景加载和游戏暂停处理。
- [Godot C# Template](https://github.com/CSharpGodotTools/Template) - 用于 C# 项目的模板，具有 UI 场景、可视化调试工具、简化的补间、实用脚本和 ENet 多人框架。
- [First Person Starter](https://github.com/Whimfoome/godot-FirstPersonStarter) - 带有第一人称控制器的模板，可以通过检查器轻松调整。
- [Godot-GameTemplate](https://github.com/nezvers/Godot-GameTemplate) - 包含所有必要内容的模板。专为像素艺术游戏而设计。
- [Godot 4 Custom Gravity](https://github.com/Ivorforce/Godot4-Custom-Gravity) - 具有行星引力的 3D 项目模板（类似超级马里奥银河）。使用有符号距离场支持任何行星形状。包括一个角色控制器和各种示例设置。
- [Godot XR Template](https://github.com/godotVR/godot-xr-template) - 多级 XR 游戏的模板。
- [Maaack's Game Template](https://github.com/Maaack/Godot-Game-Template) - 包含主菜单、选项菜单、暂停菜单、制作人员名单、场景加载器、额外工具和示例游戏场景的模板。
- [MechanicalFlower Godot Template](https://github.com/MechanicalFlower/godot-template) - 具有代码格式化、许可证跟踪、插件管理和 CI/CD 的模板。
- [Takin Godot Template](https://github.com/TinyTakinTeller/TakinGodotTemplate) - 带有保存文件系统、本地化、UI（主菜单、保存文件、选项、制作人员名单）、示例游戏场景、插件（场景和音频管理器、记录器、linter、资源查看器）、占位符（音乐、sfx）、github 工作流程和各种实用脚本的模板。

#### 戈多3

- [bendn's Godot Template](https://github.com/bend-n/godot-template) - 使用 [gpm](https://github.com/you-win/godot-package-manager) 的游戏模板，并自动进行 itch.io 推送和构建。包括入门 CLI。
- [Godot First Person Camera](https://github.com/tavurth/godot-simple-fps-camera) - 一个简单的 FPS 入门游戏，具有跳跃、移动、手电筒和带有动画的玩家角色。
- [Godot FPS Template](https://github.com/MilkAndBanana01/godot-fps-template) - Godot 引擎的免费第一人称游戏模板。
- [Godot Game Of Life](https://github.com/tavurth/godot-game-of-life) - 康威使用着色器的“生命游戏”。
- [Minimum Game](https://github.com/benmarz/minimum_game) - 模板自上而下的 2D 像素艺术游戏，具有多个房间、HUD、菜单和自动保存功能。
- [Multiplayer First Person Shooter](https://github.com/blockspacer/Godot-3.2-Multiplayer-FPS) - 多人第一人称射击游戏示例项目。
- [Multiplayer Third Person Shooter](https://github.com/jasonswearingen/TPS-MP) - 多人第三人称射击游戏示例项目。

## 插件和脚本

*让您做新事情或增强 Godot 功能的脚本。*

#### 戈多4

- [2dog](https://2dog.dev) - 在 .NET 中运行和控制 Godot，使用现有的 .NET 工具和 IDE 创建单元测试、自定义开发人员和 CI 工作流程。
- [3D Auto Collision Generator](https://github.com/ThGnommy/godot_3d_auto_collision_generator) - 一键生成多个 3D 对象的碰撞。
- [AgonesSDK](https://github.com/AndreMicheletti/godot-agones-sdk) - 用于向 Godot 添加 [Agones](https://github.com/googleforgames/agones) SDK 功能的插件。
- [AnimatedShape2D](https://github.com/Goutte/godot-addon-animated-shape-2d) - 对 CollisionShape2D 以及 AnimatedSprite2D 的帧进行动画处理。
- [AnimSheet](https://github.com/Lighar/AnimSheet) - 直接从 Godot 编辑器中的 spritesheet 自动生成 AnimationPlayer 或 AnimatedSprite2D 节点。
- [AntialiasedLine2D](https://github.com/godot-extended-libraries/godot-antialiased-line2d) - 与默认的 Godot 实现（所有渲染方法，所有平台）相比，具有更高质量的抗锯齿 Line2D 和 Polygon2D 绘图。
- [Aseprite Wizard](https://github.com/viniciusgerevini/godot-aseprite-wizard) - 用于从 Aseprite 将动画导入为 SpriteFrame 的插件。
- [Beehave](https://github.com/bitbrain/beehave) - 使您能够使用行为树创建强大的 NPC AI 系统。
- [CReverter (Composite Reverter)](https://codeberg.org/svetogam/creverter) - 基于备忘录的撤消/重做实用程序，支持组合。
- [CSConnector (Contextual Signal/Setup Connector)](https://codeberg.org/svetogam/csconnector) - 提供一个干净的界面，通过场景树动态查找、设置和连接到后代节点。
- [CSLocator (Contextual Service Locator)](https://codeberg.org/svetogam/cslocator) - 提供一个干净的接口来通过场景树注册和查找对象，例如本地化的单例。
- [Cyclops Level Builder](https://github.com/blackears/cyclopsLevelBuilder) - 用于快速原型和概念验证的关卡构建器。
- [Dialogic](https://github.com/dialogic-godot/dialogic) - 创建对话、角色和场景来显示对话。
- [Dialogue Engine](https://github.com/Rubonnek/dialogue-engine) - 简约的对话引擎适合您的 GUI 节点，并自动绘制分支对话图表以方便调试。
- [Dialogue Manager](https://github.com/nathanhoad/godot_dialogue_manager) - 一个简单但功能强大的基于文本的分支对话编辑器和运行时。 *（戈多 3 和 4）*
- [DotnetQuestSystem](https://github.com/TRUINGLol/DotnetQuestSystem) - .NET 上带有 Godot 接口的任务系统的简单实现。
- [Event Audio](https://github.com/bbbscarter/event-audio-godot) - 一个简单的基于事件的“即发即忘”音频触发系统。
- [Fast Fourier Transform (FFT)](https://github.com/tavurth/godot-fft) - GDScript 中的快速傅里叶变换。
- [FMOD GDNative](https://github.com/utopia-rise/fmod-gdnative) - 在 GDScript 中使用 FMOD 音频引擎的插件。
- [friflo ECS](https://github.com/friflo/Friflo.Engine.ECS) - 具有简单 API 的高性能 C# ECS。支持.NET、WASM/WebAssembly、Native AOT、Godot、Unity、MonoGame...*（Godot 3 和 4）*
- [func_godot](https://github.com/func-godot/func_godot_plugin) - 使用 [Quake MAP 文件格式](https://quakewiki.org/wiki/Quake_Map_Format) 导入地图，通常使用 [TrenchBroom](https://trenchbroom.github.io/) 等编辑器制作。
- [GDGIFExporter](https://github.com/jegor377/godot-gdgifexporter) - GIF 导出器完全用 GDScript 制作。
- [GdUnit4](https://github.com/MikeSchulze/gdUnit4) - Godot 单元测试框架。
- [Gedis](https://github.com/NodotProject/Gedis) - Godot 的内存中类似 Redis 的键值存储。
- [GedisQueue](https://github.com/NodotProject/GedisQueue) - 由 Gedis 支持的全面的 Godot 作业队列系统。
