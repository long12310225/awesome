## 语言

* [><>](https://esolangs.org/wiki/fish) - 一种基于堆栈的二维语言，支持反射。
* [1+](https://esolangs.org/wiki/1%2B) - 一种令人惊讶的用户友好语言，具有高度有限的指令集。
* [78](https://github.com/oatmealine/78) - 一种以数字 7 和 8 为中心的单字符命令语言。
* [ajsone](https://www.quaxio.com/ajsone) - 完全用 JSON 定义的语言。
* [AlmostBinary](https://github.com/wsdt/AlmostBinary) - 二进制代码并在任何平台上运行。
* [ArnoldC](http://lhartikk.github.io/ArnoldC) - 根据阿诺·施瓦辛格的最佳俏皮台词改编。
* [Befunge](https://esolangs.org/wiki/Befunge) - 二维语言。
* [Benedictum](https://github.com/Beneking102/benedictum) - 拉丁主题的 Brainfuck 衍生品，源代码读作祈祷。
* [Brackets](https://github.com/kvbc/brackets) - 语言是由括号组成的，很多括号。
* [Brainfuck](https://esolangs.org/wiki/Brainfuck) - 仅使用 8 个单字符命令的极简语言。
* [C Flat](https://github.com/NicksterSand/Cflat) - 音乐语言注重声音而不是记谱，并考虑到创作自由。
* [CellTail](https://github.com/mousetail/celltail) - 基于一维细胞自动机的语言，支持复杂的模式匹配语法。
* [Charcoal](https://github.com/somebody1234/Charcoal) - 用于绘制 ASCII 艺术草图的简洁语言。
* [Chef](http://www.dangermouse.net/esoteric/chef.html) - 看起来像烹饪食谱的程序。
* [Chicken](https://esolangs.org/wiki/Chicken) - 只有 1 个有效符号：“鸡”。
* [ChuckScript](https://github.com/angrykoala/chuckscript) - 基于一元的语言，其中代码是单个数字。
* [COW](https://esolangs.org/wiki/COW) - Brainfuck 变体使用 *moos* 的十二个变体作为指令。
* [DNA#](https://esolangs.org/wiki/DNA-Sharp) - 基于 DNA 结构的语言。
* [Dots](https://github.com/josconno/dots) - 使用“.”的图灵完备语言仅符号。
* [Electra](https://github.com/DolphyWind/Electra-Lang) - 像电工一样编码。
* [Emojicode](http://www.emojicode.org) - 由表情符号组成的成熟编程语言。
* [Enterprise™](https://github.com/joaomilho/Enterprise) - 非确定性的不必要的静态类型语言。
* [False](http://strlen.com/false-language) - 语言严重混淆，实现最少。
* [Fetlang](https://github.com/Property404/fetlang) - 设计得使源代码看起来像写得很糟糕的恋物癖色情作品。
* [FlipJump](https://github.com/tomhea/flip-jump) - 1条指令：翻转一点，然后跳转。最简单的OISC。
* [Folders](https://github.com/rottytooth/Folders) - 用文件夹编写的代码。
* [FreakC](https://github.com/FreakC-Foundation/FreakC) - 具有 NSFW 内容和模因的成熟编程语言。
* [Fugue](https://esolangs.org/wiki/Fugue) - 音乐作为源代码。
* [Golfscript](http://www.golfscript.com/golfscript) - 面向堆栈，旨在以尽可能少的击键次数解决问题。
* [Haifu](http://www.dangermouse.net/esoteric/haifu.html) - 设计时牢记东方哲学的核心价值观。
* [INTERCAL](http://www.catb.org/~esr/intercal) - 戏仿语言旨在与其他语言完全不同。
* [Japt](https://github.com/ETHproductions/japt) - 减少高尔夫的 JavaScript。
* [Jelly](https://github.com/DennisMitchell/jellylanguage) - 基于 J. 的高尔夫语言
* [JellyScript](https://github.com/nguyenphuminh/Jellyscript) - 以 Brainfuck 为基础，搭配美味的果冻。
* [JSFuck](https://github.com/aemkei/jsfuck) - 基于 JavaScript 原子部分的深奥编程风格。
* [Lazy K](https://tromp.github.io/cl/lazy-k.html) - 最小的、图灵完备的纯函数语言，没有额外的功能。
* [legit](https://morr.cc/legit) - 程序完全由 git 存储库的图表而不是其内容定义。
* [LolCode](http://lolcode.org) - 带有基于 lolcat meme 表达式的关键字的语言。
* [Malbolge](https://esolangs.org/wiki/Malbolge) - 以第八层地狱命名，旨在尽可能难以编程。
* [Mindfck](https://github.com/angrykoala/mindfck) - 转译为 Brainfuck 的高级语言。
* [Monicelli](https://github.com/esseks/monicelli) - 使用类似意大利语的乱码进行编程。
* [Ni](https://github.com/DeybisMelendez/ni) - 基于 Brainfuck 并提供更多有用的说明。
* [Omgrofl](https://esolangs.org/wiki/Omgrofl) - 语法类似于互联网俚语。
* [Ook!](http://www.dangermouse.net/esoteric/ook.html) - 基于 Brainfuck 的语言，其中的命令是猩猩单词。
* [Orca](https://hundredrabbits.itch.io/orca) - Livecoding 环境可快速创建程序音频音序器。
* [Piet](http://www.dangermouse.net/esoteric/piet.html) - 程序看起来像使用颜色作为代码的抽象绘画的语言。以彼特·蒙德里安的名字命名。
* [RASEL](https://github.com/Nakilon/rasel) - Fungeoid 对程序空间、浮点精度没有限制，并且对堆栈具有一定的随机访问能力。
* [reMorse](http://esolangs.org/wiki/reMorse) - 莫尔斯风格的语言。
* [Rockstar](https://github.com/dylanbeattie/rockstar) - 专为创建同时也是歌词的计算机程序而设计。
* [Shakespeare](http://shakespearelang.sourceforge.net) - 语言里的代码看起来像是莎士比亚戏剧。
* [Schoenberg](https://tomeraberba.ch/schoenberg) - MIDI 深奥编程语言。
* [Spooky](https://spookylang.com) - 最可怕的编程语言：jack_o_lantern：：skull：。
* [Suzy](https://github.com/gvx/suzy) - 3D 语言的灵感源自 Befunge。
* [Technologic](https://esolangs.org/wiki/Technologic) - 程序所用的语言看起来像 Daft Punk 歌曲 Technologic 的歌词。
* [Thue](https://github.com/jcolag/Thue) - *Matrioshka* 语言基于半 Thue 语法的非确定性字符串重写。
* [Tru](https://github.com/sungwoncho/tru) - 一种基于堆栈的语言，带有无前缀括号。
* [Turtle](https://github.com/TypeMonkey/Turtle) - 使用长期存储来进行内存使用。
* [Unary](https://esolangs.org/wiki/Unary) - 基于 Brainfuck 的语言，所有命令都是 0。
* [Unary Except Every Zero Is Replaced with the Title of This Programming Language or, Alternately, Is Replaced with the Smallest Counter-Example to the Goldbach Conjecture. Compilers and Interpreters Only Have to Implement the Former Option](https://esolangs.org/wiki/Unary_Except_Every_Zero_Is_Replaced_with_the_Title_of_This_Programming_Language_or,_Alternately,_Is_Replaced_with_the_Smallest_Counter-Example_to_the_Goldbach_Conjecture._Compilers_and_Interpreters_Only_Have_to_Implement_the_Former_Option) - 一元的导数，其中每个零都替换为正在讨论的编程语言的标题。
* [Unlambda](http://www.madore.org/~david/programs/unlambda) - 模糊的函数式编程。
* [Velato](http://velato.net) - 使用 MIDI 文件作为源的语言，程序由音符的音高和顺序定义。
* [Vigil](https://github.com/munificent/vigil) - 只有 Vigil 真正警惕不允许无法通过编程规范的代码。
* [Watson](https://github.com/genkami/watson) - 基于堆栈的图灵不完整配置文件。
* [文言 (Wenyan)](http://wenyan-lang.lingdong.works) - 严格遵循中国古典文学的语法和语气。
* [Whenever](http://www.dangermouse.net/esoteric/whenever.html) - 随时随地做事的语言。
* [Whitespace](http://web.archive.org/web/20150623025348/http://compsoc.dur.ac.uk/whitespace) - 仅使用白色字符（空格、制表符和换行符）。
* [WordLang](https://github.com/WilliamRagstad/WordLang) - 一种基于类似意义的语法并产生输出的语言。
* [Z](https://esolangs.org/wiki/Z) - 仅使用字母 z。
* [ZOMBIE](https://www.dangermouse.net/esoteric/zombie.html) - 专为死灵法师设计的本质邪恶的多线程语言。

## 算法

* [πfs](https://github.com/philipl/pifs) - 基于Pi的存储系统，通过在pi中搜索匹配序列来存储数据。
* [Stacksort](https://gkoberger.github.io/stacksort) - 通过在 Stack Overflow 中搜索答案并执行代码片段直到排序，对列表进行排序。
