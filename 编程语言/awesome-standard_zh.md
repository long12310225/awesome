## 标准

- **[标准](https://www.npmjs.com/package/standard)** - JavaScript 标准样式。一种风格统治一切！

## 使用统计

- **[使用标准的 npm 包（完整列表）](https://github.com/feross/standard-packages/blob/master/all.json)** - 使用 `standard` 的包列表
- **[标准下载图表 (npm-stat.com)](http://npm-stat.com/charts.html?package=standard&author=&from=&to=)** - npm 注册表下载计数
- **[顶级 npm 软件包（要点）](https://gist.github.com/feross/e0882df2fe673d6ce064)** - standard 是 npm 上的顶级软件包之一

## 标准使用的包

- **[eslint](https://www.npmjs.com/package/eslint)** - 用于 JavaScript 的基于 AST 的模式检查器
- **[standard-engine](https://www.npmjs.com/package/standard-engine)** - 标准的内容，提取到可重用的包中。建立你自己的！
- **[eslint-config-standard](https://www.npmjs.com/package/eslint-config-standard)** - 标准 eslint 规则，作为可共享的配置
- **[eslint-config-standard-jsx](https://www.npmjs.com/package/eslint-config-standard-jsx)** - JSX 的标准 eslint 规则
- **[eslint-plugin-standard](https://www.npmjs.com/package/eslint-plugin-standard)** - 标准自定义 eslint 规则（不是 eslint 核心的一部分）

## 自动代码格式化程序

- `standard --fix` - 自动格式化内置于`standard`！
- **[prettier-standard](https://www.npmjs.com/package/prettier-standard)** - 将 [prettier](https://github.com/prettier/prettier) 配置为标准规则的格式
- **[“unix 命令”要点](https://gist.github.com/watson/453fc63cace521fcdadc)** - 搜索和替换 unix 命令列表，以帮助使节点存储库符合“标准”要求

## 编辑器插件

- 每个主要的文本编辑器都有一个插件或内置对“标准”的支持。请参阅支持的编辑器的[完整列表](https://github.com/standard/standard#are-there-text-editor-plugins)。
- Atom：**[linter-js-standard](https://atom.io/packages/linter-js-standard)** - linter
- Atom：**[linter-js-standard-engine](https://atom.io/packages/linter-js-standard-engine)** - linter
- Atom：**[标准格式化程序](https://atom.io/packages/standard-formatter)** - 自动修复程序
- Atom：**[standardjs-snippets](https://atom.io/packages/standardjs-snippets)** - 代码片段
- 括号：**[brackets-standard](https://github.com/ishamf/brackets-standard/)** - linter
- Emacs：请参阅[自述文件](https://github.com/standard/standard#emacs)
- Sublime Text：**[SublimeLinter-contrib-standard](https://packagecontrol.io/packages/SublimeLinter-contrib-standard)** - linter
- Sublime Text：**[StandardFormat](https://packagecontrol.io/packages/StandardFormat)** - 自动修复程序
- Visual Studio Code：**[vscode-standard](https://marketplace.visualstudio.com/items?itemName=standard.vscode-standard)** - linter 和自动修复程序
- Vim：请参阅[自述文件](https://github.com/standard/standard#vim)
- Visual Studio Code：**[vscode-standardjs-snippets](https://marketplace.visualstudio.com/items?itemName=capaj.vscode-standardjs-snippets)** - 代码片段
- Visual Studio Code：**[vscode-react-standard](https://marketplace.visualstudio.com/items?itemName=TimonVS.ReactSnippetsStandard)** - 代码片段
- WebStorm（PhpStorm、IntelliJ、RubyMine、JetBrains 等）：内置支持。请参阅[自述文件](https://github.com/standard/standard#webstorm-phpstorm-intellij-rubymine-jetbrains-etc)

## 漂亮的终端输出（记者）

- **[snazzy](https://www.npmjs.com/package/snazzy)** - 将 JavaScript 标准样式格式化为时尚（即时髦）输出
- **[standard-tap](https://www.npmjs.com/package/standard-tap)** - 将 JavaScript 标准样式格式化为 TAP 输出
- **[standard-json](https://www.npmjs.com/package/standard-json)** - 将 JavaScript 标准样式输出格式化为 JSON 数组
- **[standard-reporter](https://www.npmjs.com/package/standard-reporter)** - 通过unix管道的feross/standard报告器，许多选项
- **[standard-summary](https://www.npmjs.com/package/standard-summary)** - 将输出显示为错误列表，按计数排序

## 构建工具

# 受到“标准”的启发

- **[standard-markdown](https://www.npmjs.com/package/standard-markdown)** - Lint markdown 文件中的所有 javascript 代码块
- **[stylelint-config-standard](https://www.npmjs.com/package/stylelint-config-standard)** - stylelint 的标准可共享配置。
- **[standard-sass](https://www.npmjs.com/package/standard-sass)** - Sass & Scss & Css 标准样式。一种风格统治一切！
- **[wemake-frontend-styleguide](https://github.com/wemake-services/wemake-frontend-styleguide)** - 基于标准工具的 JavaScript & TypeScript & Vue & JsDoc & Scss 样式指南。非常严格！

## 叉子

- **[semistandard](https://www.npmjs.com/package/semistandard)** - 标准，带分号
- **[uber-standard](https://www.npmjs.com/package/uber-standard)** - 由 [Uber](https://www.uber.com/) 团队分叉
- **[videojs-standard](https://www.npmjs.com/package/videojs-standard)** - 由 [video.js](https://github.com/videojs/video.js) 团队分叉
- **[yodlr-standard](https://www.npmjs.com/package/yodlr-standard)** - 由 [yodlr](https://getyodlr.com/home) 团队分叉
- **[happiness](https://www.npmjs.com/package/happiness)** - 标准定制，让 @JedWatson 快乐
- **[doublestandard](https://www.npmjs.com/package/doublestandard)** - 标准，带有更多分号
- **[xo](https://www.npmjs.com/package/xo)** - “JavaScript 幸福风格”
- **[jsw](https://www.npmjs.com/package/jsw)** - “规范不关心分号，你也不应该”
- **[奥巴马](https://www.npmjs.com/package/obama)** - 与奥巴马一起前进并标准化
- **[aStandard](https://npmjs.com/package/a-standard)** - 为 babel 和 es7 制定的更严格的标准
- **[standard-own](https://npmjs.com/package/standard-own)** - 标准，但具有更多自定义和单独的规则。
- **[standard-focus](https://www.npmjs.com/package/standard-focus)** - 一次关注一个错误。
