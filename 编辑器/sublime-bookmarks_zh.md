# 崇高文本书签 [![Awesome](https://cdn.rawgit.com/sindresorhus/awesome/d7305f38d29fed78fa85652e3a63e154dd8e8829/media/badge.svg)](https://github.com/sindresorhus/awesome)

> [Sublime Text](http://www.sublimetext.com/) 是一个跨平台的文本和源代码编辑器，具有Python应用程序编程接口。其功能可通过插件进行扩展。大多数扩展包都有免费软件许可证，并且是社区构建和维护的。 — [维基百科](https://en.wikipedia.org/wiki/Sublime_Text)

这个小项目遵循 [GitHub 社区趋势](https://github.com/sindresorhus/awesome)，以方便的结构良好的集合的形式聚合特定主题的最重要的书签。在这里，您可以找到 Sublime Text 的教程和学习材料、用于编码和文本编辑的通用扩展以及按使用情况分组的专用扩展。

这个列表不应该包含所有 Sublime Text 插件，因为我们已经有 [Package Control](https://packagecontrol.io/)。它旨在成为帮助设置工作环境的起点，或者检查新的扩展以使您现有的 Sublime 设置更加出色。您可能会在列表中到处找到★。它代表编辑选择。非常感谢分享和请求请求！

**[分叉并编辑](https://github.com/dreikanter/sublime-bookmarks/edit/master/README.md)**（或[提出一些建议](https://github.com/dreikanter/sublime-bookmarks/issues/new)来添加）。

## 扩展

仅限 Alive 项目，支持 Sublime Text 版本 3。

### 通用型

- [All Autocomplete](https://github.com/alienhard/SublimeAllAutocomplete) - 扩展默认自动完成功能以在所有打开的文件中查找匹配项。
- [ApplySyntax](https://github.com/facelessuser/ApplySyntax) - 一个插件，允许检测和应用否则可能无法正确检测的文件语法。例如，扩展名为 .rb 的文件通常是 Ruby 文件，但当在 Rails 项目中找到它们时，它们可能是 RSpec 规范文件、Cucumber 步骤文件、Ruby on Rails 文件（控制器、模型等）或只是普通的 Ruby 文件。
- [EditorConfig](https://github.com/sindresorhus/editorconfig-sublime) - .editorconfig 支持。
- [File Navigator](https://github.com/csch0/SublimeText-File-Navigator) - 键盘友好的文件导航器和管理器。
- [FindKeyConflicts](https://github.com/skuroda/FindKeyConflicts) - 识别冲突的键映射。
- [GitGutter](https://github.com/jisaacks/GitGutter) - 一个在装订线中查看 git diff 的插件。
- [GitSavvy](https://packagecontrol.io/packages/GitSavvy) - git 和 GitHub 集成。
- [Glue](https://github.com/chrissimpkins/glue) - Sublime Text 2 和 3 的跨平台、可扩展插件，可将您最喜欢的编辑器连接到您的 shell。
- [LSP](https://packagecontrol.io/packages/LSP) - Sublime Text 3 的语言服务器协议支持为您提供 IDE 功能。
- [Origami](https://github.com/SublimeText/Origami) - 随心所欲地分割窗口！创建新窗格、删除窗格、在窗格之间移动和克隆视图。
- [PackageResourceViewer](https://github.com/skuroda/PackageResourceViewer) - 查看和编辑 Sublime 包资源。有助于扩展和主题开发。
- [PlainTasks](https://packagecontrol.io/packages/PlainTasks) - 固执己见的待办事项清单。
- [Sidebar Enhancements](https://github.com/titoBouzout/SideBarEnhancements) - 增强了文件和文件夹侧边栏的操作。
- [StyleToken](https://github.com/vcharnahrebel/style-token) - 允许用不同的颜色突出显示某些文本片段（类似于 Notepad++“样式标记”功能）。
- [Sublime Alignment](https://github.com/wbond/sublime_alignment) - 用于对齐多行和多个选择的简单键绑定。
- [Sublime Worksheet](https://github.com/jcartledge/sublime-worksheet) - 适用于 JavaScript、PHP、Ruby 等的内联 REPL。
- [SublimeFileBrowser](https://github.com/aziz/SublimeFileBrowser) - 抛弃侧边栏，像专业人士一样使用键盘在普通选项卡中浏览文件！此外，[Dired](https://en.wikipedia.org/wiki/Dired) 允许您使用 ST 提供的强大文本编辑功能修改目录结构。
- [SublimeLinter](https://github.com/SublimeLinter/SublimeLinter3) - 一个提供 linting 代码框架的插件。无论您使用什么语言编写代码，SublimeLinter 都可以帮助您编写更干净、更好、更无错误的代码。 SublimeLinter 旨在为用户提供最大的灵活性和可用性，并为 linter 作者提供最大的简单性。
- [SublimeREPL](https://github.com/wuub/SublimeREPL) - 在 ST2 中运行解释器（Clojure、CoffeeScript、F#、Groovy、Haskell、Lua、MozRepl、NodeJS、Python、R、Ruby、Scala、shell 或自行配置）。
- [Sync Settings](https://github.com/mfuentesg/SyncSettings) - 保持 Sublime Text 配置同步的跨平台解决方案
- [Synesthesia](https://github.com/dariusf/synesthesia) - 一个用漂亮的颜色突出显示文本的插件。
- [Trailing Spaces](https://github.com/SublimeText/TrailingSpaces) - 突出显示尾随空格并立即将其删除。

### JavaScript 简介

- [Bump](https://github.com/yavorsky/Bump) - 用于保持 package.json 依赖项最新的插件。在包悬停时的简洁工具提示中显示最新的 npm/yarn 依赖项版本。
- [CoffeeScript](https://github.com/Xavura/CoffeeScript-Sublime-Plugin) - 语法突出显示和检查、命令、快捷方式、片段、编译等。
- [JavaScript Enhancements](https://github.com/pichillilorenzo/JavaScriptEnhancements) - 该插件提供智能 javascript 自动完成、实时错误以及许多有关创建、开发和管理 javascript 项目（例如 Cordova、Ionic、React 等）的功能。它在底层使用了 [Flow](https://github.com/facebook/flow)（来自 Facebook 的 javascript 静态类型检查器）。
- [JsFormat](https://github.com/jdc0589/JsFormat) - JavaScript 格式化插件。它使用来自 http://jsbeautifier.org 的命令行/python-module JavaScript 格式化程序来格式化整个 js 或 json 文件或选定的部分。
- [TernJS](https://github.com/ternjs/tern_for_sublime) - JavaScript 类型推理引擎集成。

### 主题

- [Ayu](https://github.com/dempfi/ayu) - 简单的主题，明亮的色彩，提供三种版本——深色、幻景和浅色，适合全天舒适的工作。
- [Ciapre](https://github.com/vinhnx/Ciapre.tmTheme/) - 赏心悦目的 Sublime Text/TextMate 配色方案。
- [Cobalt2](https://github.com/wesbos/cobalt2) - 功能齐全的 Sublime Text 蓝色主题。
- [Comfort](https://github.com/devanshdalal/Comfort) - Sublime Text 的一个精彩主题！
- [Cyanide](https://github.com/lefoy/cyanide-theme) - 一个最小的黑暗主题。
- [Flatland](https://github.com/thinkpixellab/flatland) - Sublime Text 2 和 3 的简单主题和随附的配色方案。它主要源自 Soda，Soda 是 Sublime 任何自定义主题开发的正确起点。
- [Material theme](https://github.com/equinusocio/material-theme) - 这个主题带来了Material Design的视觉语言。 http://equinsuocha.io/material-theme
- [Predawn](https://github.com/jamiewilson/predawn) - Sublime Text 和 Atom 的深色界面和语法主题。 http://jamiewilson.io/predawn
- [Soda](http://buymeasoda.github.io/soda-theme/) - 深色和浅色自定义 UI 主题。
- [Spacegray](http://kkga.github.io/spacegray/) - Sublime Text 2/3 的一组自定义 UI 主题。这都是关于炒作和最小化的。有不同的口味以及相应的 Base16 配色方案。 http://kkga.github.io/spacegray
